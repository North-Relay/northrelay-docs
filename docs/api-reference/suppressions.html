Suppressions API

Manage suppression lists to prevent sending to unsubscribed or invalid email addresses.

Suppressions API

The Suppressions API manages addresses that should not receive emails. Suppressions prevent delivery to recipients who have bounced, complained, or explicitly unsubscribed. NorthRelay automatically adds addresses to the suppression list on hard bounces and spam complaints; you can also manage suppressions manually via the API.

How Suppressions Work

When you send an email, NorthRelay checks the recipient address against your suppression list before queuing it. If the address is suppressed, the email is silently dropped and not counted against your monthly quota.

Suppression sources:

SourceTriggerRemovable
bounceHard bounce (5xx SMTP response)Yes, via API
complaintSpam complaint via feedback loopYes, via API
manualAdded via API or dashboardYes, via API
unsubscribeRecipient clicked unsubscribe linkYes, via API

Suppression Groups

Suppression groups enable category-based unsubscribes. Instead of globally suppressing an address, you can suppress it only for specific email categories (e.g., "marketing" but not "transactional").

Delete Suppression Group

Deleting a group also removes all suppressions in that group. Those addresses become eligible for delivery again.

Using Groups When Sending

Include a categories array in your Send Email request. NorthRelay checks the recipient against suppression groups that match those categories:

If alice@example.com is suppressed in the "Product Updates" group, the email is dropped. If she is only suppressed in the "Marketing" group, the email is delivered.
