Authentication

How to create, manage, and use NorthRelay API keys for authenticating requests.

Authentication

All NorthRelay API requests require authentication. The primary method is Bearer token authentication using API keys.

API Key Formats

NorthRelay issues two types of API keys, distinguished by prefix:

PrefixEnvironmentBehavior
nr_live_ProductionSends real emails through your configured delivery pool
nr_test_TestSimulates sending without delivering; charges no quota

Both key types authenticate against the same endpoints. Use nr_test_ keys during development and integration testing to avoid consuming your monthly quota.

Creating an API Key

  1. Navigate to Dashboard > Settings > API Keys
  2. Click Create API Key
  3. Enter a descriptive name (e.g., "Production Backend", "Staging CI")
  4. Select the key scope (see Key Scopes below)
  5. Click Generate
  6. Copy the key immediately -- it is shown only once

The key is a long random string prefixed with nr_live_ or nr_test_:

Key Scopes

API keys can be scoped to limit their permissions:

ScopePermissions
full_accessAll API operations (send, domains, templates, contacts, webhooks, analytics)
sendingSend emails, use templates, validate addresses
read_onlyList and read resources; no create, update, or delete operations

Select the narrowest scope that meets your use case. A key with sending scope cannot manage domains or webhooks.

Session Authentication

Dashboard endpoints also accept NextAuth.js session cookies. This is used automatically when you interact with the API from the NorthRelay dashboard. Session auth is not recommended for server-to-server integrations -- use API keys instead.

Security Best Practices

  1. Never commit API keys to version control. Store them in environment variables or a secrets manager.

  2. Use separate keys per environment. Create distinct keys for production, staging, and development.

  3. Rotate keys regularly. Revoke and regenerate keys every 90 days or after any suspected exposure.

  4. Use the narrowest scope. A backend that only sends emails should use a sending-scoped key, not full_access.

  5. Use nr_test_ keys for development. Avoid consuming production quota or accidentally sending real emails during testing.

  6. Restrict server-side only. Never expose API keys in client-side JavaScript, mobile apps, or public repositories.

  7. Monitor key usage. Review API key activity in Dashboard > Settings > API Keys to detect unauthorized use.

  8. Revoke compromised keys immediately. If a key is leaked, revoke it in the dashboard and generate a replacement.
