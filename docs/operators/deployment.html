Deployment Guide

Deploy NorthRelay's sovereign SMTP infrastructure on your own VPS.

Prerequisites

Before deploying NorthRelay, ensure you have:

RequirementMinimumRecommended
VPS RAM4 GB8 GB
CPU Cores24
Disk40 GB SSD80 GB SSD
OSUbuntu 22.04+Ubuntu 24.04 LTS
Docker24.0+Latest
Docker Composev2.20+Latest
Domain1 domain2 domains (corporate + system)

You also need:

  • A public IPv4 address with PTR record configured
  • Ports 25, 443, 587, 993 open in your firewall
  • DNS control for your domain(s)

Architecture Overview

NorthRelay runs as a Docker Compose stack with the following services:

ServiceImagePurpose
appNext.js (custom build)Dashboard and API
postgrespostgres:15-alpinePrimary database
redisredis:7-alpineCaching and rate limiting
stalwartstalwart/mail-serverOutbound SMTP relay and queue
miniominio/minioObject storage (templates, attachments)
cronalpine/curlScheduled task runner
nginxnginx:alpineReverse proxy with TLS termination

For the full production topology with two-pool routing, see Stalwart Setup.

DNS Records

At minimum, configure these DNS records for your domain:

TypeNameValuePurpose
Ayourdomain.comYOUR_VPS_IPDashboard
Amail.yourdomain.comYOUR_VPS_IPMail server
MXyourdomain.commail.yourdomain.comInbound mail
TXTyourdomain.comv=spf1 ip4:YOUR_VPS_IP -allSPF
TXT_dmarc.yourdomain.comv=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.comDMARC

For DKIM DNS records, see Configuration - DKIM.
