Diagnostic Tools

NorthRelay provides three built-in diagnostic tools to help you identify and resolve delivery issues without contacting support. Access them from Dashboard > Tools.

Pre-Flight Check

The Pre-Flight Check validates your sending configuration before you send a single email. It tests DNS records, SMTP connectivity, and authentication in one pass.

What It Tests

CheckDescriptionPass Criteria
SPF RecordVerifies your domain has a valid SPF TXT record that includes NorthRelayRecord contains include:_spf.north-pipe.ca
DKIM RecordVerifies the DKIM CNAME record points to NorthRelay's signing keyCNAME resolves to the expected NorthRelay target
DMARC RecordChecks for a DMARC policy on your domain_dmarc.yourdomain.com TXT record exists
MX RecordsValidates your domain's MX configuration (for inbox domains)MX records resolve correctly
SMTP ConnectivityTests that NorthRelay can establish an SMTP connection for your domainConnection succeeds within timeout
AuthenticationValidates your API key and sending permissionsAPI key is valid and has send permission

How to Use

  1. Go to Dashboard > Tools > Pre-Flight Check
  2. Enter the domain you want to test (must be added to your account)
  3. Click Run Check
  4. Review the results -- each check shows pass, fail, or warning with a specific message

Interpreting Results

  • All green: Your configuration is correct. You are ready to send.
  • Yellow warnings: Non-critical issues (e.g., DMARC policy is none instead of reject). Sending will work but deliverability may not be optimal.
  • Red failures: Critical issues that will prevent delivery or cause authentication failures. Fix these before sending.

Each failed check includes an action field explaining exactly what to fix and a link to the relevant documentation.

Blocklist Monitor

The Blocklist Monitor checks your domain and sending IPs against 13 industry-standard DNS-based Real-time Blocklists (RBLs). Being listed on a blocklist can severely impact your deliverability.

Provider Coverage

The monitor checks 13 consolidated providers organized into four tiers by importance:

Tier 1: Industry Standard (3 providers)

These are the blocklists that major mailbox providers actively check. A listing here has the most impact.

ProviderTypeDescription
Spamhaus ZENIP-basedCombined Spamhaus blocklist (SBL + XBL + PBL). The most widely used blocklist.
BarracudaIP-basedBarracuda Networks reputation system. Used by many enterprise mail gateways.
SpamCopIP-basedCrowdsourced spam reporting. Listings are time-based and auto-expire.

Tier 2: Compromised Hosts & Network Threats (6 providers)

These detect compromised systems, botnets, and network-level abuse.

ProviderTypeDescription
CBLIP-basedComposite Blocking List. Detects IPs sending spam from bots/malware.
PSBLIP-basedPassive Spam Block List. Trap-based detection.
InvaluementIP-basedDetects snowshoe spam and bulk senders with poor practices.
BackscattererIP-basedDetects IPs generating backscatter (bounces to forged senders).
0spamIP-basedZero-spam project. Algorithmic detection.
SpamRatsIP-basedDetects IPs on dynamic/residential ranges sending spam.

Tier 3: Reputation & Behavior Systems (3 providers)

These track sending reputation and behavioral patterns over time.

ProviderTypeDescription
HOSTKARMAIP-basedJunk Email Filter. Reputation-based scoring.
MailspikeIP-basedReal-time reputation tracking with historical data.
SORBSIP-basedSpam and Open Relay Blocking System. Multiple zone checks.

Tier 4: Domain & URI Reputation (1 provider)

Checks domain reputation rather than IP addresses.

ProviderTypeDescription
SURBL MultiDomain-basedCombined phishing, malware, and abuse domain list. Checks domains in email content.

How to Use

  1. Go to Dashboard > Tools > Blocklist Monitor
  2. Enter a domain or IP address to check
  3. Click Check
  4. Results appear grouped by tier in collapsible accordion sections

Result Statuses

StatusMeaningAction
CleanNot listed on this blocklistNo action needed
ListedYour IP or domain is on this blocklistFollow the provider's delisting process
WarningListed on a lower-priority blocklistMonitor and consider delisting
Rate LimitedThe blocklist provider rate-limited our queryTry again in a few minutes
OfflineThe blocklist provider is not respondingNot your issue -- check again later
TimeoutQuery timed out (3-second limit)Likely a provider issue -- retry
ErrorUnexpected error during lookupRetry; if persistent, contact NorthRelay support

If You Are Listed

  1. Identify the cause. Review your Reputation Monitoring dashboard for incidents.
  2. Fix the underlying issue. High bounce rates and spam complaints are the most common causes of blocklisting.
  3. Request delisting. Each blocklist provider has a delisting process:
    • Spamhaus: https://check.spamhaus.org
    • Barracuda: https://www.barracudacentral.org/lookups
    • SpamCop: Listings auto-expire after reports stop (typically 24-48 hours)
  4. Re-check. After delisting, run the Blocklist Monitor again to confirm.

Technical Details

  • All 13 providers are queried in parallel
  • Each query has a 3-second timeout
  • No registration or paid subscription required -- all providers are freely accessible
  • Queries use standard DNS-based blocklist lookup (reverse IP + blocklist zone)

Connectivity Doctor

The Connectivity Doctor tests the full SMTP delivery chain from NorthRelay to a destination mail server. Use it to diagnose why emails to a specific domain are failing.

What It Tests

CheckDescriptionWhat It Reveals
DNS ResolutionResolves MX records for the destination domainWhether the destination domain has valid mail servers
SMTP ConnectionConnects to the destination MX server on port 25Whether the mail server is reachable
TLS HandshakeAttempts STARTTLS upgradeWhether the connection can be encrypted
EHLO ResponseSends EHLO and checks server capabilitiesWhat extensions the server supports (SIZE, PIPELINING, etc.)
AuthenticationTests that NorthRelay's sending IP is acceptedWhether the destination server accepts mail from NorthRelay

How to Use

  1. Go to Dashboard > Tools > Connectivity Doctor
  2. Enter the destination domain you are having trouble delivering to (e.g., gmail.com)
  3. Click Test Connection
  4. Review the step-by-step results

Interpreting Results

Each step shows a pass/fail status with timing information:

  • DNS Resolution failed: The destination domain has no MX records or its DNS is misconfigured. This is a problem on the recipient's side.
  • SMTP Connection failed: The destination mail server is unreachable. Could be a firewall, server outage, or network issue.
  • TLS Handshake failed: The destination server does not support STARTTLS or has a certificate issue. Emails may still deliver without encryption depending on your settings.
  • EHLO rejected: The destination server rejected NorthRelay's HELO/EHLO. This could indicate IP-based blocking.
  • Authentication rejected: The destination server is rejecting mail from NorthRelay's IP. Check the Blocklist Monitor to see if the IP is listed.
