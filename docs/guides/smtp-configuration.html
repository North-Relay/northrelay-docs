SMTP Configuration

This guide covers NorthRelay's SMTP settings in detail, including authentication, TLS configuration, and connection testing.

SMTP Settings

SettingValue
Hostsmtp.north-pipe.ca
Port (STARTTLS)587 (recommended)
Port (Implicit TLS)465
AuthenticationRequired (LOGIN or PLAIN)
UsernameYour NorthRelay account email
PasswordYour API key (nr_live_xxxxxxxxxxxx)
EncryptionTLS 1.2 or later

Use port 587 with STARTTLS unless your application only supports implicit TLS (port 465).

Authentication

NorthRelay requires authentication for all SMTP connections. Your credentials are:

  • Username: The email address you used to create your NorthRelay account (e.g., you@example.com)
  • Password: An active API key from Dashboard → API Keys (e.g., nr_live_xxxxxxxxxxxx)

NorthRelay supports both LOGIN and PLAIN SASL mechanisms. Most SMTP clients will negotiate the mechanism automatically.

Creating an API Key

  1. Log in to app.northrelay.ca
  2. Go to Dashboard → API Keys
  3. Click Create API Key
  4. Copy the key immediately — it is shown only once
  5. Use this key as your SMTP password

You can create multiple API keys for different applications. Revoke any key instantly from the same page.

TLS/SSL

Port 587 — STARTTLS (Recommended)

The connection starts unencrypted, then upgrades to TLS via the STARTTLS command. This is the industry standard for SMTP submission.

Port 465 — Implicit TLS

The connection is TLS-encrypted from the start. Use this if your application does not support STARTTLS, or if port 587 is blocked on your network.

TLS Requirements

  • Minimum TLS version: 1.2
  • TLS 1.0 and 1.1 are not supported
  • Server Name Indication (SNI) is supported

Pool-Specific Behavior

NorthRelay uses two IP pools for sending. Your pool is determined by your plan tier:

PoolTiersEHLO Hostname
SharedFree, Starter, Microshared.north-pipe.ca
IsolatedPro, Scale, Enterprisesovereign.north-pipe.ca

This is handled automatically by NorthRelay's routing infrastructure. You do not need to configure anything differently — all users connect to the same smtp.north-pipe.ca host and NorthRelay routes your email through the appropriate pool.

Troubleshooting

Connection timeout

  • Verify your server can reach smtp.north-pipe.ca on port 587 or 465
  • Some cloud providers (AWS EC2, GCP, Azure) block outbound SMTP by default — check your provider's documentation
  • Try the alternate port (465 if 587 is blocked, or vice versa)

Authentication failure (535)

  • Username must be your NorthRelay account email, not a custom sending domain address
  • Password must be an active API key starting with nr_live_
  • Check for trailing whitespace when copying the API key

TLS errors

  • Ensure your client supports TLS 1.2 or later
  • On port 587, your client must send the STARTTLS command before authenticating
  • On port 465, your client must initiate TLS immediately upon connection
  • Update your system's CA certificate bundle if you see certificate verification errors

"Relay access denied" (554)

  • Confirm you are authenticated before sending
  • Ensure the From address uses a domain you own or a NorthRelay subdomain
