Automatic Retries

The SDK retries failed requests with exponential backoff:

  • Retries on 5xx server errors and network timeouts
  • Does not retry on 4xx client errors
  • Backoff: 1s, 2s, 4s ... capped at max_retry_delay (default 10s)
  • Rate limit responses (429) are retried after the retry_after interval
